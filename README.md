# Shell Snippet

一个美观、易用的 Shell 命令小抄 (Cheat Sheet) Web 应用。欢迎参与提交脚本和共建！

在线体验！[点这里](https://shell.samhou.moe)

## 功能特性

* **⚡️ 快速搜索**：实时过滤查找所需的命令。
* **🎨 深色模式**：支持跟随系统设置自动切换，也可手动切换 (🌓)。
* **📱 响应式设计**：完美适配桌面和移动端，采用 Grid 布局。
* **📝 交互式代码块**：
  * **行号显示**：方便阅读长脚本。
  * **一键复制**：快速复制命令。
  * **在线编辑**：点击 "Edit" 可临时修改命令参数（如替换变量）后再复制。
* **📖 详细说明**：支持为每个命令添加标题和详细解释。

## 贡献

欢迎提交自己的 shell 小抄！跟着我操作：

* Fork 这个仓库
* 创建一个新的 branch（随意命名）
* 在 docs 里面增加一个新的 txt 文件，命名规范 `命令涉及到项目/程序/脚本-执行的操作-特定操作系统.txt`。不针对特定操作系统可不写。比如 `fail2ban-install.txt`。不要包含空格和特殊字符！你可以用 `-` 连起来。
* 第一行写标题，然后写简介（可选），然后用单独一行 `---` 隔开标题简介和你的 shell 命令（参考下方的 DEV）
* 所有脚本应该**用英语书写**，便于搜索命令时不用切输入法。提交中文脚本也可以，但是合并前后**我会帮你翻译为英文**。
* 提交 PR。标题命名为 `docs: Add xxx`，xxx 改成你第一行的标题，或者一个更加简洁的表述。
* 等待审核。如果 CI 构建出现问题或 Review 时建议修改，你可以在你的 branch 上推送修复 commit。

## DEV 开发

### 1. 添加小抄

在 `docs/` 目录下创建 `.txt` 文件。

格式说明：

* **第一行**：命令标题（显示在列表和详情页标题）。
* **第二行及之后**（可选）：详细解释，支持多行。
* **分隔符**：使用 `---` 单独一行分隔说明和代码。
* **剩余部分**：具体的 Shell 命令或脚本。

**示例 (`docs/acme-install.txt`)**:

```text
Install acme
Install acme to the machine
---
curl https://get.acme.sh | sh -s email=you@example.com
source ~/.bashrc
```

### 2. 构建项目

运行构建脚本将 `docs/` 下的文本文件转换为前端资源：

```bash
node build.js
```

### 3. 运行

构建完成后，`public/` 目录即为生成的静态网站。你可以使用任何静态文件服务器打开它。

```bash
npm start # 快速启动静态服务器
```

## 目录结构

* `docs/`: 存放小抄源文件 (.txt)。
* `src/`: 前端源码 (HTML, CSS, JS)。
* `public/`: 构建生成的静态网站（不要直接修改）。
* `build.js`: 构建脚本。
